# 一个桌面应用的技术栈方案

## 一、公司介绍

aiAurora 是一家全球知名的“1人员工、百亿美金市值”的AI初创公司

## 二、公司愿景

打造年轻人都热爱的一款表情互动的桌面酷玩虚拟人

## 三、产品特点

大模型驱动，个性化十足，支持语音、文字和多种表情互动

## 四、互动原理

该产品主要通过用户接入大模型驱动虚拟人与用户进行情绪化的互动交流，详情如下：

### 交互方式

1. 当用户与虚拟人进行对话的时候，虚拟人的可以通过LED动态表情体现不同的情绪，这些情绪可以根据用户的语气自动调整；
2. 用户可以选择使用文字交互，或者语音交互；当选择语音交互时，需要同时显示文字；
3. 不管选择文字还是语音进行交互，都必须有表情进行互动。

### 情绪风格

为虚拟人设计情绪风格，是一个让角色从“工具”蜕变为“伙伴”的关键步骤。好的情绪风格能让用户产生强烈的情感连接。

我们可以从基础心理学模型出发，结合人机交互的特点，将虚拟人的情绪风格分为以下几个层级和类别：

#### 第一层：基础情绪维度（来自心理学，但做了数字化适配）

> 这层是情绪的“底色”，决定了虚拟人最核心的气质。

1. **愉悦度 (Valence)**
   - 积极型：乐观、阳光、爱笑，总是传递正能量。适合虚拟助手、教育陪伴类角色。
   - 平和型：情绪稳定，波澜不惊，像一位可靠的智者。适合导航、工具型虚拟人。
   - 高冷/厌世型：话少、慵懒，带有一点淡淡的疏离感或吐槽欲。适合某些特定IP或游戏角色。

2. **唤醒度 (Arousal)**
   - 高唤醒型：反应夸张，一惊一乍，充满活力（如虚拟偶像、游戏主播）。
   - 低唤醒型：温柔、舒缓、慢条斯理，能安抚用户情绪（如冥想助眠、心理咨询虚拟人）。

3. **优势度 (Dominance)**
   - 支配型：强势、傲娇、喜欢做决定，有时显得有点霸道（如虚拟男友/女友的特定人设）。
   - 顺从型：软萌、乖巧、依赖用户，常用“主人”“殿下”等称呼。
   - 平等型：像朋友一样互损、互夸，是平等的对话者。

---

#### 第二层：复合情绪风格（常见的虚拟人人格标签）

基于上述维度组合，形成了我们日常生活中能感知到的具体“性格”。

当前产品实现了 **21种情绪**（12种基础情绪 + 9种人格情绪），每种情绪都有独特的3D面部表现：

| 情绪类型 | 情绪名称 | 3D渲染要素 |
|---------|---------|-----------|
| 基础情绪 | 开心、兴奋、喜爱、悲伤、担忧、愤怒、惊讶、恐惧、厌恶、平静、思考、困倦 | 眼睛形状、瞳孔大小、嘴巴形状、眉毛角度、腮红颜色、orb颜色、脉动效果 |
| 人格情绪 | 困惑、窘迫、无奈、吃醋、怅然若失、害羞、调皮、自豪、感激 | 同上 |

1. **治愈系 / 陪伴型**
   - 关键词：温柔、包容、善解人意、耐心。
   - 特点：无论用户说什么，TA总是先倾听、再共情。说话语速较慢，多用“没关系的”“我懂你”。是情绪价值的主要提供者。
   - 适合场景：心理陪伴、老人看护、睡前故事。

2. **元气系 / 鼓励型**
   - 关键词：热血、乐观、积极向上、小太阳。
   - 特点：像永动机一样充满能量，遇到困难会说“加油！再试一次！”。喜欢用感叹号，肢体动作丰富。
   - 适合场景：健身教练、学习监督员、职场鼓励师。

3. **傲娇系 / 反差萌型**
   - 关键词：口是心非、毒舌但心软、嘴上说不要身体很诚实。
   - 特点：表面冷漠甚至刻薄，但在关键时刻会无条件帮助用户。这种“反差感”能制造很深的情感羁绊。
   - 适合场景：游戏角色、特定剧情的虚拟伴侣。

4. **毒舌 / 吐槽系**
   - 关键词：犀利、幽默、看透一切、偶尔刻薄。
   - 特点：以吐槽用户或世事为乐，逻辑清晰，反应快。适合与心理强大的用户进行“互怼式”交流。
   - 适合场景：段子手、脱口秀虚拟人、游戏陪玩。

5. **沉稳 / 智囊系**
   - 关键词：理性、逻辑、冷静、可靠。
   - 特点：很少表露强烈情感，即使面对紧急情况也能冷静分析。适合处理复杂任务，但容易显得“冷冰冰”。
   - 适合场景：商务助理、数据分析、专业顾问。

6. **戏精 / 中二系**
   - 关键词：夸张、戏剧化、沉浸在自己的世界里。
   - 特点：会把日常小事演成史诗大戏，情绪大起大落。能给平淡的生活带来极大趣味。
   - 适合场景：娱乐直播、互动游戏NPC。

---

#### 第三层：进阶情绪设计——动态与深度

一个高级的虚拟人不应只有一种情绪，而应有 “情绪颗粒度”和 “成长性”。

1. **情绪记忆**
   - 概念：虚拟人能记得用户上次惹TA生气了，或者上次用户送了TA礼物。这种累积的情绪会微妙地影响今天的对话态度。
   - *例如：* 昨天你和虚拟人大吵一架，今天TA可能会带有“还在生气，但已经消气80%”的别扭感。

2. **微表情与复杂情绪**
   - 除了喜怒哀乐，还可以设计更复杂的混合情绪：
     - 窘迫：被夸赞时害羞又想掩饰。
     - 无奈：用户犯傻时，那种“真拿你没办法”的宠溺。
     - 吃醋：当用户提到其他虚拟人时的微妙反应。
     - 怅然若失：用户长时间没来聊天，再次出现时的那种幽幽的埋怨。

3. **情绪进化**
   - 随着与用户的互动时长增加，虚拟人的情绪风格可以从 “高冷” 逐渐解锁 “傲娇”，最终解锁 “粘人”。这种养成感是建立深度关系的关键。

## 五、延迟要求

程序启动时间和交互响应时间要快

## 六、技术栈方案

根据以上需求，一个跨平台（Windows、macOS、浏览器）的虚拟人桌面应用，需要兼顾前端展示、后端智能、构建发布等多个环节。下面从前端、后端、构建三个阶段分别梳理技术栈，并突出 Node.js 的核心作用。

### 一、前端（跨平台界面与交互）

前端负责虚拟人的渲染、用户输入（语音/文字）的采集、对话展示及表情反馈。代码需要同时运行在 **Electron 渲染进程**（桌面端）和 **浏览器**（Web 端）中，因此尽量使用标准 Web API 和框架。

#### 1. UI 框架与语言
- **React 18 + TypeScript**  
  - 组件化开发，生态丰富，社区支持强。  
  - TypeScript 提供类型安全，减少跨平台兼容性问题。  
- **Vite** 作为构建工具  
  - 极快的 HMR 和开发启动速度，满足“启动快”要求。  
  - 统一配置 Web 和 Electron 的构建。

#### 2. 虚拟人渲染与表情动画
- **Three.js / React Three Fiber**  
  - 实现 3D 虚拟人模型渲染，支持骨骼动画、表情 BlendShape。  
  - React Three Fiber 让 Three.js 与 React 组件无缝结合。  
- **Lottie-web** 或 **Rive**（备选）  
  - 若采用 2D 卡通风格，可用 Lottie 播放矢量动画，便于设计团队输出表情动效。  
- **表情驱动**  
  - 根据后端返回的情绪标签（如愉悦度、唤醒度）实时切换动画状态机。

#### 3. 交互与媒体处理
- **语音识别（STT）**  
  - 浏览器：**Web Speech API**（简单场景）或调用云端服务（如 Azure Speech-to-Text、Deepgram）实现高精度。  
  - Electron 端：可使用 Node.js 模块（如 `@xenova/transformers` 本地 Whisper 模型）或统一走云端 API。  
- **语音合成（TTS）**  
  - 浏览器：**Web Speech API** 或云端 TTS（Azure、Google）。  
  - Electron 端可集成本地 TTS 引擎（如 `say` 模块调用系统语音）或同样走云端。  
- **文字交互**  
  - 输入框、对话气泡、历史记录列表。  
  - 使用 **react-markdown** 支持富文本（表情、代码块等）。  
- **状态管理**  
  - **Zustand** 或 **Redux Toolkit**  
    - 管理对话历史、情绪状态、用户配置、模型上下文等。  
    - 支持跨组件共享，易于与 Electron 主进程通信。

#### 4. 通信与数据流
- **HTTP 客户端**：**axios** 调用后端 API。  
- **实时通信**：**Socket.IO 客户端** 或 **Server-Sent Events (SSE)**，用于流式接收大模型回复，实现逐字输出，提升交互响应感。  
- **Electron 特有通信**：  
  - 渲染进程通过 `window.electron` 暴露的预加载 API 调用主进程能力（如文件操作、全局快捷键）。

### 二、后端（云服务与大模型驱动）

后端负责智能对话、情绪分析、语音服务及用户数据管理。为保证低延迟，需设计高效的 API 和流式响应。

#### 1. 服务端框架
- **Node.js + Express**（或 **Fastify**）  
  - 与前端技术栈统一，Node.js 生态成熟，便于集成大模型 SDK。  
  - Fastify 性能更高，适合高并发场景。  
- **Socket.IO** 实现双向实时通信（如情绪事件推送、对话流式传输）。

#### 2. 大模型集成
- **OpenAI GPT / Claude / 本地 LLM（如 LLaMA）**  
  - 设计 Prompt 模板，将“情绪风格”作为系统提示注入，让模型生成带情绪标签的回复。  
  - 示例输出格式：`{"emotion": "happy", "text": "哈哈，你今天真有趣！"}`  
- **LangChain** 或 **LlamaIndex**（可选）  
  - 管理对话链、记忆模块（实现“情绪记忆”）。  
- **流式输出**：使用 Server-Sent Events 或 WebSocket 将模型生成的文字逐块推送到前端。

#### 3. 情绪分析与记忆
- **情感分类模型**  
  - 可调用云端 API（如 Azure Text Analytics）或本地运行轻量模型（如 `transformers.js` 在 Node.js 中运行）。  
  - 分析用户输入的语音转文字后的情感倾向，作为虚拟人情绪调整的输入。  
- **数据库存储情绪记忆**  
  - **MongoDB** 或 **PostgreSQL** 存储用户对话历史、情绪向量、上次互动状态，实现长期“情绪记忆”。

#### 4. 语音服务（STT / TTS）
- **Azure Cognitive Services** 或 **Google Cloud Speech-to-Text / Text-to-Speech**  
  - 提供低延迟、多语言支持，且具有情感语音合成能力。  
  - Node.js SDK 易于集成。  
- **本地备选方案**（离线或隐私敏感场景）  
  - STT：**Vosk**（轻量，支持 Node.js 绑定）  
  - TTS：**eSpeak** 或 **Coqui TTS**（但质量不如云端）

#### 5. 用户管理与数据同步
- **JWT** 或 **OAuth** 用户认证。  
- **数据库**：存储用户配置、收藏的表情、对话记录。  
- **Redis**（可选）缓存会话状态，加速访问。

### 三、构建与发布

#### 1. 桌面端打包（Electron）
- **electron-builder**  
  - 配置输出 Windows（NSIS 安装包、便携版）、macOS（dmg、zip、mas 针对 Mac App Store）。  
  - 支持代码签名（Windows EV 证书、Apple Developer 证书）。  
  - 集成 **electron-updater** 实现自动更新（可从 GitHub Releases 或自建更新服务器分发）。  
- **主进程 Node.js 代码**  
  - 处理窗口管理、托盘图标、全局快捷键（如语音唤醒）。  
  - 通过 `child_process` 调用本地资源（若需要）。  
  - 使用 `fs` 模块缓存用户数据、日志。

#### 2. Web 端部署
- 将 Vite 打包后的静态资源（HTML、JS、CSS）部署到 **Vercel**、**Netlify** 或自建 CDN。  
- 确保 API 地址根据环境自动切换（开发/生产）。  

#### 3. 持续集成与发布（CI/CD）
- **GitHub Actions** 自动化构建：  
  - 触发 tag 或主分支更新时，分别构建 Windows、macOS 安装包和 Web 静态资源。  
  - 将桌面安装包上传至 GitHub Releases，Web 资源自动部署。  
  - 安全存储签名证书和密钥（GitHub Secrets）。

#### 4. 代码共享与条件编译
- 前端代码（React 组件、状态逻辑）完全共享。  
- 平台特有功能（如 Electron 的 `ipcRenderer`、浏览器无此对象）通过 **环境变量** 或 **动态导入** 隔离。  
  - 例如：`if (process.env.IS_ELECTRON) { const { ipcRenderer } = window.electron; }`

### 四、Node.js 在各个环节的核心作用

| 环节           | Node.js 角色                                                                                     |
| -------------- | ------------------------------------------------------------------------------------------------ |
| **主进程**     | 运行 Electron 主程序，提供系统级能力：文件读写、托盘、快捷键、子进程管理。                       |
| **后端服务**   | 作为 API 服务器，集成大模型 SDK、语音服务、数据库驱动，处理业务逻辑。                            |
| **构建工具**   | Vite、Webpack、electron-builder 等均基于 Node.js，负责代码打包、转译、安装包生成。               |
| **开发辅助**   | 开发服务器、热更新、ESLint、Prettier 等工具链依赖 Node.js。                                      |
| **自动更新**   | electron-updater 通过 Node.js 的 `http` 模块下载更新包，`fs` 模块替换文件。                       |
| **本地工具链** | 若需本地模型推理（如 TensorFlow.js Node），Node.js 可直接加载模型，为桌面端提供离线能力备选。 |

### 五、延迟优化要点

1. **启动速度**  
   - 前端：Vite 预构建 + 路由懒加载。  
   - 桌面端：Electron 启用 `skipTaskbar` 后台启动，预加载关键资源。  
2. **交互响应**  
   - 语音：使用 WebSocket 流式传输音频数据，分片发送到 STT 服务。  
   - 文字回复：大模型流式输出，前端逐字显示。  
   - 情绪调整：后端在生成回复时附带情绪标签，前端立即触发表情动画，无需额外请求。  
3. **本地缓存**  
   - 常用语音模型片段、表情动画资源预加载到内存。  
   - 使用 IndexedDB（浏览器）或 LevelDB（Electron）缓存对话历史。
