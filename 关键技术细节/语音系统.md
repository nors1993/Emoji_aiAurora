# 语音交互链路（按下录音 → 语音识别 → LLM → TTS 播放）
**完整流程**：

```
用户按下 [Hold to Speak] 按钮
  → AudioCapture.start() 
    → getUserMedia + AudioContext 16kHz PCM（Pulse Code Modulation，脉冲编码调制）
      → 音频数据发送至 FunASR WebSocket
        → VAD 检测语音结束
          → ASR 返回识别文本
            → 文本填入 Chat 输入框，自动发送
              → sendToLLM() 流式请求
                → 逐 token 显示在 Chat 中
                  → 完整回复到达
                    → 情绪检测(detectEmotion)
                      → Avatar 切换表情
                        → TTSClient.speak(text, emotion)
                          → Qwen3-TTS 流式返回音频
                            → AudioContext 播放
```

**关键设计**：

1. **非阻塞结构**：ASR 和 TTS 都基于事件驱动（callbacks），不阻塞 UI。
2. **音频缓冲**：`pcmBufferRef` 累积 PCM 帧，支持断网重传。
3. **TTS 中断**：`ttsClient.stop()` 可随时打断当前语音（用户再次说话时）。
4. **语音模式与文字模式并存**：即使启用语音，聊天窗口仍显示文字，满足"语音交互、文字同步"的产品要求。

# FunASR
**架构**：

```
浏览器麦克风 → AudioCapture (16kHz PCM) → WebSocket → FunASR Server
                                                                  ↓
浏览器 Chat ← WebSocket JSON ←--- FunASR 解码 ← SenseVoice 编码
```

**AudioCapture 实现**：

- 通过 `navigator.mediaDevices.getUserMedia()` 获取麦克风流。
- `AudioContext` 设置采样率 16kHz（ASR 标准输入格式）。
- `ScriptProcessorNode` 处理音频块（每帧 4096 samples）。
- Float32 → Int16 PCM 转换：`pcm[i] = s < 0 ? s * 0x8000 : s * 0x7fff`

**VAD（WebRTC Voice Activity Detection）**：

- 在 `funasr_server.py` 中使用 `webrtcvad` 库。
- 30ms 帧（16kHz × 0.03s = 480 samples = 960 bytes）。
- 连续 3 帧有语音 → `speech_started` 事件。
- 连续 15 帧静音 → `speech_end` → 触发识别结果返回。
- 这种方式避免了"静音期间反复识别"的问题，只有完整语音段才返回结果。

**传输格式**：

- 二进制帧（PCM raw data）。
- JSON 控制消息（`{type: "final", text: "..."}`、`{type: "speech_started"}`）。

# Qwen3-TTS
**完整链路**：

```
LLM 回复内容 + 检测到的情绪(Emotion)
  → emotionToInstruct(emotion) → 自然语言指令字符串
    → TTS API 请求体中的 instruct 字段
      → Qwen3-TTS 模型推理 → 带情绪语调的 WAV 音频
        → 浏览器 AudioContext 播放
```

**映射实现**（`types/index.ts` 中的 `emotionToInstruct`）：

```typescript
const EMOTION_INSTRUCT: Record<Emotion, string> = {
  happy: '用开心愉悦的语气说',
  sad: '用悲伤难过的语气说',
  angry: '用生气不满的语气说',
  excited: '用兴奋激动的语气说',
  love: '用温柔爱意的语气说',
  playful: '用调皮逗趣的语气说',
  // ... 每种情绪对应一个自然语言指令
}

export function emotionToInstruct(emotion: Emotion): string {
  return EMOTION_INSTRUCT[emotion] || '用自然的语气说'
}
```

**为什么用自然语言指令而非数值参数**：
Qwen3-TTS 模型在训练时学习了"自然语言描述 → 语音韵律"的映射，而非传统的风格编号。这使得：

1. 指令语义丰富（"开心愉悦"包含了快乐、愉悦、积极等多维信息）。
2. 自然语言指令比单一数值维度更直观、更可控。
3. 与 21 种情绪的 PAD 量化值互补：PAD 是"分析引擎"的量化标准，instruct 是"生成引擎"的自然语言接口。

**流式播放**：

- SSE 流式逐句返回：`{index: 0, audio: "base64...", text: "你好呀"}`
- 浏览器端边解码边播放，减少首句延迟。
- `cleanTextForTTS()` 预处理：去除 emoji、括号注释、多余空格，避免 TTS 朗读特殊字符。
